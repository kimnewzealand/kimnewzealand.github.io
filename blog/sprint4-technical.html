Introduction to Problem Solving

Generally problems are simple, straightforward to solve, or absolute head scratchers.
The sprint 4 curriculum a full stack of new JavaScript challenges, and problems to solve.
In JavaScript you need to create boxes to put values in. These boxes are called objects and have labels for the values. During the challenges I encountered a problem of forgetting to refer to the labels. I learnt to use a technique called pseudocode which is as it sounds. It's fake code so a machine can't interpret it. It is like writing a script outline for a play, but I now find it an elegant and descriptive way to structure code before writing in code.

What was the problem? What problem solving techniques did you use? How did you feel throughout the process? What did you learn?

Pseudocode

Confidence 3/5

I have written comments above lines of code. However I didn't realise until now that there is a name for this non technical step by step description of code, called pseudocode. Pseudocode can also be defined as:

an informal high-level description of the operating principle of a computer program or other algorithm. It uses the structural conventions of a programming language, but is intended for human reading rather than machine reading.

One approach is to start with simple statements with the goal of the code with statements. Then rewrite to represent the structure of the code. See this example. Then these pseudocode statements can be rewritten in the code language syntax you are using.
